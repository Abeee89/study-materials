# Software Requirements Specification (SRS) & Development Plan
## Project: Antigravity — AI-Powered Web-Based Electrical Engineering Learning Platform

---

## 1. Project Overview & Scope

### 1.1 Introduction
**Antigravity** is an interactive, full-stack web-based educational ecosystem tailored for vocational and undergraduate students studying fundamental electrical engineering and electronics. The platform bridges theoretical concepts with active, practical application by embedding high-fidelity interactive simulation suites directly into a modern learning management system (LMS). 

### 1.2 Core Value Proposition
Every feature within the system is unified under an artificial intelligence umbrella driven by **OpenRouter API** free-tier models. The AI does not merely act as an isolated chatbot but serves as a persistent, context-aware pedagogical companion. It adapts to student workflows, reads live user states across chapters, and continuously parses performance metrics from formative evaluations to build individualized learning vectors.


### 1.3 Strict Operational Constraints
* **Framework Infrastructure:** Next.js 14+ (App Router), React Server Components (RSC).
* **Database Engine:** PostgreSQL (Deployed via free-tier cloud instances like Neon.tech or Supabase).
* **AI Provider:** OpenRouter API utilizing exclusively free inference models (e.g., `google/gemma-2-9b-it:free`, `meta-llama/llama-3-8b-instruct:free`).
* **Deployment & Hosting Target:** Vercel (Hobby Free Tier).
* **Cost Cap:** $0.00 USD total operational cost for base infrastructure.

## 2. Technical Stack Architecture

The application layout leverages a modern, decoupled serverless architecture built on top of Vercel and cloud-native managed PostgreSQL.

| Layer | Technology Selected | Rationale for Selection | Free-Tier Boundaries |
| :--- | :--- | :--- | :--- |
| **Frontend Framework** | Next.js 14+ (App Router) | Server-side rendering (SSR) for static content pages; optimized streaming API routes for real-time AI tokens. | Unlimited deployment limits on Vercel Hobby tier. |
| **Styling & UI Components** | Tailwind CSS + Lucide Icons | Utility-first compilation ensures zero-runtime CSS overhead and blazing-fast performance. | Fully open-source / built-in. |
| **State Management** | React Context + Local Hooks | Local state (`useState`, `useReducer`) handles heavy client-side mathematical math updates in circuit engines without server round-trips. | Client-side execution. |
| **Authentication & Auth** | NextAuth.js (Auth.js v5) | Lightweight, secure Session-cookie handler natively integrated with Next.js middleware for role isolation. | Open-source. |
| **Database Engine** | PostgreSQL (Neon / Supabase) | Supports native JSONB storage patterns for quizzes and provides full transactional integrity. | 0.5 GB storage, pool connections included. |
| **Database Gateway** | Prisma ORM | Type-safe schema generation, automatic migration tracking, and intuitive JavaScript client generation. | Free open-source tool. |
| **AI Inference Orchestration** | OpenRouter Serverless API | Consolidates open-source LLMs under a single, unified unified completion API without credit card bindings. | $0.00 rate-limited free token endpoints. |

### 3.1 Extended Prisma Schema Spec Configuration

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

enum UserRole {
  STUDENT
  TEACHER
}

enum MediaType {
  PDF
  VIDEO
  TEXT
}

model User {
  id            Int            @id @default(autoincrement())
  name          String
  email         String         @unique
  passwordHash  String
  role          UserRole       @default(STUDENT)
  createdAt     DateTime       @default(now())
  
  // Relations
  managedClasses Classes[]     @relation("TeacherClasses")
  enrollments   Enrollment[]
  assessments   Assessment[]
  evaluations   AiEvaluation[]
}

model Classes {
  id          Int          @id @default(autoincrement())
  name        String
  classCode   String       @unique
  teacherId   Int
  createdAt   DateTime     @default(now())
  
  // Relations
  teacher     User         @relation("TeacherClasses", fields: [teacherId], references: [id], onDelete: Cascade)
  enrollments Enrollment[]
}

model Enrollment {
  id         Int      @id @default(autoincrement())
  studentId  Int
  classId    Int
  enrolledAt DateTime @default(now())

  // Relations
  student    User     @relation(fields: [studentId], references: [id], onDelete: Cascade)
  class      Classes  @relation(fields: [classId], references: [id], onDelete: Cascade)

  @@unique([studentId, classId])
}

model Chapter {
  id         Int          @id @default(autoincrement())
  title      String
  sortOrder  Int          @unique
  subs       SubChapter[]
}

model SubChapter {
  id          Int       @id @default(autoincrement())
  chapterId   Int
  title       String
  contentType MediaType
  contentBody String?   @db.Text
  mediaUrl    String?
  sortOrder   Int

  // Relations
  chapter     Chapter   @relation(fields: [chapterId], references: [id], onDelete: Cascade)
  quiz        Quiz?
}

model Quiz {
  id           Int          @id @default(autoincrement())
  subChapterId Int          @unique
  title        String
  questions    Json         // Array format: [{question: string, options: string[], correct: string}]

  // Relations
  subChapter   SubChapter   @relation(fields: [subChapterId], references: [id], onDelete: Cascade)
  assessments  Assessment[]
}

model Assessment {
  id          Int      @id @default(autoincrement())
  studentId   Int
  quizId      Int
  score       Int
  studentAns  Json         // Key-value snapshot of student selections
  completedAt DateTime @default(now())

  // Relations
  student     User     @relation(fields: [studentId], references: [id], onDelete: Cascade)
  quiz        Quiz     @relation(fields: [quizId], references: [id], onDelete: Cascade)
}

model AiEvaluation {
  id                    Int      @id @default(autoincrement())
  studentId             Int
  strengths             String   @db.Text
  weaknesses            String   @db.Text
  suggestions           String   @db.Text
  recommendedSubChapters Int[]    // Primitive Int Array tracking needed lesson nodes
  evaluatedAt           DateTime @default(now())

  // Relations
  student               User     @relation(fields: [studentId], references: [id], onDelete: Cascade)
}
```

### 6.1 Serverless API Architecture Route Sample (`/api/ai/evaluate/route.ts`)
```typescript
import { NextResponse } from 'next/server';

export async function POST(req: Request) {
  try {
    const { studentId, quizId, score, studentAnswers, quizContext } = await req.json();

    const systemPrompt = `You are an expert AI Electrical Engineering Professor inside the Antigravity learning platform. 
Analyze the student's performance data completely. You must output your evaluation exactly in the following raw text block structure:
[STRENGTHS]
Summarize clear concepts mastered.
[WEAKNESSES]
Identify exact gaps or mathematical errors found.
[SUGGESTIONS]
Provide precise actionable study advice.`;

    const userPrompt = `Student ID: ${studentId}
Quiz Reference: ${quizContext.title}
Score Achieved: ${score}%
Detailed Response Matrix: ${JSON.stringify(studentAnswers)}
Expected Questions Setup: ${JSON.stringify(quizContext.questions)}`;

    const openRouterResponse = await fetch("https://openrouter.ai/api/v1/chat/completions", {
      method: "POST",
      headers: {
        "Authorization": `Bearer ${process.env.OPENROUTER_API_KEY}`,
        "Content-Type": "application/json",
        "HTTP-Referer": "https://antigravity-edu.vercel.app", 
      },
      body: JSON.stringify({
        model: "google/gemma-2-9b-it:free",
        messages: [
          { role: "system", content: systemPrompt },
          { role: "user", content: userPrompt }
        ],
        temperature: 0.3
      })
    });

    const aiData = await openRouterResponse.json();
    const rawAiText = aiData.choices[0].message.content;

    // Inside full system architecture: Parse structural tokens and execute a Prisma INSERT directly into the Database.
    return NextResponse.json({ rawEvaluation: rawAiText });
  } catch (error) {
    return NextResponse.json({ error: "Pipeline Failure" }, { status: 500 });
  }
}
```

## 7. Operational Roles & Security Layers (Feature 5)

Data separation is enforced globally at the Next.js Edge Layer using route-matching middleware architecture.

```

                                            |
                                            v
                                   /middleware.ts Engine
                                            |
                       +--------------------+--------------------+
                       |                                         |
                       v                                         v
               Starts with /student                     Starts with /teacher
                       |                                         |
            [ Check Token Role ]                      [ Check Token Role ]
                       |                                         |
            +----------+----------+                   +----------+----------+
            |                     |                   |                     |
            v                     v                   v                     v
       Role == STUDENT     Role != STUDENT       Role == TEACHER     Role != TEACHER
            |                     |                   |                     |
            v                     v                   v                     v
       [ Allow Access ]    [ Redirect /login ]   [ Allow Access ]    [ Redirect /dashboard ]
```

### 7.1 Security Access Rules Map
* **Student Route Domain (`/dashboard/student/*`):** Grant access to learning nodes, local simulations, self-quizzes, and personal historical AI assessment records.
* **Teacher Analytics Domain (`/dashboard/teacher/*`):** Restrict access completely to `role === 'TEACHER'`. Route runs targeted PostgreSQL filters to view aggregated data grids:
```sql
SELECT u.name, u.email, ae.strengths, ae.weaknesses, ae.evaluated_at
FROM users u
JOIN enrollments e ON u.id = e.student_id
JOIN ai_evaluations ae ON u.id = ae.student_id
WHERE e.class_id = :targetClassId
ORDER BY ae.evaluated_at DESC;
```
